**Зачем это было сделано? Чему научился в процессе выполнения задания?**
Пакеты core, dto и rest обычно используются для организации кода и разделения
логики на различные области ответственности:

1) Пакет core (или “основной”) содержит центральную логику приложения. В нем обычно находятся:
- Бизнес-логика: Это основной функционал приложения, который выполняет задачи, с которыми работает приложение.
Например, обработка данных, выполнение вычислений, взаимодействие с базой данных и т.д.
- Сервисные классы: Классы, которые содержат логику для обработки запросов и выполнения
задач на основе бизнес-правил.
- Утилиты: Вспомогательные классы и методы, которые могут быть использованы в различных частях приложения
(например, логирование, обработка ошибок, общие алгоритмы и т.д.).

2) Пакет dto содержит DTO (Data Transfer Object) классы. Эти объекты используются для передачи данных
между слоями приложения или между приложением и внешними системами (например, клиент-серверное взаимодействие).
DTO обычно не содержат бизнес-логики, а только хранят данные.
- Простота: DTO содержит только данные (поля), необходимые для выполнения конкретной операции.
- Обмен данными: Используются для передачи данных между слоями приложения
(например, между слоем сервиса и слоем представления) или через сеть (например, при передаче данных через API).

3) Пакет rest обычно содержит компоненты, отвечающие за реализацию REST API (Representational State Transfer).
Это веб-сервис, который предоставляет данные и функциональность приложения через HTTP-протокол.
В этом пакете могут быть:
- Контроллеры (Controllers): Классы, которые обрабатывают HTTP-запросы, выполняют необходимые действия (например, вызов
бизнес-логики из слоя core) и возвращают ответы клиенту.
- Маршруты (Endpoints): Методы, помеченные аннотациями, такие как @GetMapping, @PostMapping и т.д., которые
отвечают на запросы от клиента.
- Сериализация/десериализация данных: Преобразование объектов данных (DTO) в формат JSON или обратно.
