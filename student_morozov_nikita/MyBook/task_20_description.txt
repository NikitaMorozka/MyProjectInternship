**Зачем это было сделано? Чему научился в процессе выполнения задания?**
Stateless объекты не хранят состояния, то есть не имеют полей,
которые могут изменяться при вызове методов. Это означает, что каждый вызов метода такого объекта
1) Работа ведется только с переданными параметрами в метод
2) Не зависит от предыдущих вызовов или потоков (т.к. не хранит состояние объекта) -> не может изменять его состояние
